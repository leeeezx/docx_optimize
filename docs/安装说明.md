# 全局 `.dotm` 安装说明

## 一、适用场景

本文档面向拿到项目分发包的最终用户，目标是完成：

`安装 DocxOptimize.dotm -> 让 Word 全局加载 -> 运行项目脚本`

---

## 二、分发包建议内容

| 文件 | 作用 |
|---|---|
| `DocxOptimize.dotm` | 项目自带宏模板 |
| `scripts/packaging/install_dotm.ps1` | 安装模板 |
| `scripts/packaging/uninstall_dotm.ps1` | 卸载模板 |
| `scripts/packaging/test_dotm.ps1` | 自检模板和宏 |
| `READMe.md` | 项目总览 |

---

## 三、安装前检查

| 检查项 | 是否必须 | 说明 |
|---|---:|---|
| Windows | 是 | 当前脚本依赖 Word COM |
| Microsoft Word | 是 | 模板需要在 Word 中加载 |
| 启用宏 | 是 | 宏被禁用时无法执行 |
| MathType | 按需 | 完整公式流程需要 |
| 关闭所有 Word 窗口 | 是 | 安装和卸载前都建议关闭 |

---

## 四、安装步骤

安装链路：

`关闭 Word -> 运行安装脚本 -> 启动 Word -> 启用宏 -> 运行自检`

### 1. 执行安装脚本

```powershell
pwsh -ExecutionPolicy Bypass -File scripts\packaging\install_dotm.ps1
```

若模板不在默认位置，可显式指定：

```powershell
pwsh -ExecutionPolicy Bypass -File scripts\packaging\install_dotm.ps1 -TemplatePath .\dist\DocxOptimize.dotm
```

### 2. 脚本会做什么

| 步骤 | 动作 |
|---|---|
| 1 | 检查 Word 是否仍在运行 |
| 2 | 定位待安装的 `DocxOptimize.dotm` |
| 3 | 复制到 `%AppData%\Microsoft\Word\STARTUP` |
| 4 | 若已有旧版本，则先备份 |
| 5 | 尝试解除文件锁定 |
| 6 | 输出后续自检提示 |

### 3. 第一次启动 Word

启动 Word 后，若出现安全提示，按以下原则处理：

| 提示 | 操作 |
|---|---|
| 启用内容 / 启用宏 | 允许 |
| 文件来自互联网，已阻止 | 先右键文件属性，执行“解除锁定” |
| 受信任位置限制 | 将模板所在目录加入受信任位置，或让管理员放行 |

---

## 五、自检步骤

自检链路：

`检查模板文件 -> 检查宏可见 -> 检查脚本可调用`

### 1. 命令行自检

```powershell
pwsh -ExecutionPolicy Bypass -File scripts\packaging\test_dotm.ps1
```

### 2. Word 图形界面自检

| 检查项 | 操作 |
|---|---|
| 宏是否出现 | 打开 Word，按 `Alt + F8` |
| 工程是否加载 | 按 `Alt + F11`，查看 `Project (DocxOptimize.dotm)` |
| MathType 宏是否存在 | 宏列表中查找 `MTCommand_ConvertEqns` |

---

## 六、使用方式

### 1. 完整流程

执行链：

`OLE -> MathML -> OMML -> 输出新文件`

```powershell
python scripts\math_ops\03run_equation_pipeline.py input.docx --out output.docx
```

### 2. 仅执行项目自带宏

```powershell
pwsh -ExecutionPolicy Bypass -File scripts\math_ops\02convert_equation_format_MathML_to_OMML.ps1 -DocxPath input.docx -OutPath output.docx
```

---

## 七、卸载方式

```powershell
pwsh -ExecutionPolicy Bypass -File scripts\packaging\uninstall_dotm.ps1
```

卸载脚本不会直接删除模板，而是将 Startup 目录中的模板改名备份，便于回滚。

---

## 八、常见问题

| 现象 | 原因 | 处理办法 |
|---|---|---|
| 提示未检测到 `DocxOptimize.dotm` | 模板未安装到 Startup | 重新执行安装脚本 |
| Word 找不到 `PlainMathMLToEquation` | 模板未加载或宏被禁用 | 重启 Word，启用宏，再执行自检 |
| 第 1 步找不到 `MTCommand_ConvertEqns` | MathType 未安装或未正确集成 | 检查 MathType 和 Word 插件 |
| 安装后仍失败 | 文件被锁定或不是最新版本 | 解除锁定，重新安装覆盖 |
